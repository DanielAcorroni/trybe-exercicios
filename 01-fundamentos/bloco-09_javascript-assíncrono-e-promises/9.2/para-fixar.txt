01- O que é um código que é executado de modo assíncrono? Qual é a diferença disso para um código que é executado de modo síncrono?
R: Um código executado de modo assíncrono continua sua execução enquanto o leito passa para as linhas de baixo enquanto um código executado de modo síncrono depende da sua leitura e execução para passar para o próximo código.

02- Quando você tem que enfileirar várias callbacks, que problema surge?
R: O problema do callback hell.

03- Por que as Promises são uma forma de se resolver esse problema?
R: Pois ela pode tratar da necessidade de um código só ser lido se o anterior for concluído de forma mais legível.

04- Quando você vai se comunicar com uma API , tal comunicação deve ser síncrona ou assíncrona? Lembre-se de que o serviço ao qual você está se conectando pode demorar muito ou pouco para dar retorno, pode estar fora do ar etc.
R: Assíncrona.

05- Dada a resposta anterior, o que é fetch e para que ele serve?
R: Serve para buscar e resgatar em formato binário alguma informção de uma API.
